# Quant Backtester Design

**Date**: 2026-02-10
**Status**: Approved

## 1. 目标

- **辅助投资决策**：在现有 DCF 估值基础上，加入买卖时机信号
- **回测验证**：验证投资假设是否有效（如 DCF 低估股票是否真的跑赢大盘）
- **学习探索**：边学边做，理解量化交易的完整流程

## 2. 核心设计决策

| 维度 | 决策 | 理由 |
|------|------|------|
| 投资风格 | 基本面选股 + 技术面择时，中长期为主 | 与现有 DCF 系统互补 |
| 回测框架 | 自研 pandas-based，不用第三方库 | 与现有代码无缝衔接，完全透明，适合学习 |
| 数据范围 | 5-10 年（2016-2026） | 覆盖牛熊周期，SEC 数据从 ~2010 开始 |
| 股票池 | Phase 1: 已有 21 只 → Phase 2: Screener 动态选股 | 先跑通再扩展 |
| 仓位管理 | 风险平价（按波动率倒数分配） | 控制单只股票对组合的冲击 |
| DCF 角色 | 作为多因子系统中的一个因子，不单独依赖 | DCF 对假设敏感，需要与其他信号交叉验证 |

## 4. 因子体系（6 个因子，3 大类）

### 4.1 价值类

**F1: DCF 折扣率**
- 公式：`(DCF 公允价值 - 当前市价) / DCF 公允价值`
- 折扣越大 → 越低估 → 得分越高
- 数据来源：现有 `dcf_builder.py` 输出

**F2: 相对估值分位**
- 当前 P/E、P/S 在自身过去 5 年历史中的百分位
- 分位越低 → 越便宜 → 得分越高
- 数据来源：yfinance 历史数据

### 4.2 动量/技术类

**F3: 趋势确认**
- 价格在 200 日均线上方 → 趋势向上
- RSI < 70 → 未超买
- 两个条件都满足 → 通过确认（布尔信号）

**F4: 价格动量**
- 近 6 个月收益率排名（在股票池内排名）
- 排名越高 → 动量越强 → 得分越高

### 4.3 周期类

**F5: 宏观周期（regime 信号）**
- 收益率曲线（10Y-2Y 利差）：正 → 扩张，倒挂 → 衰退信号
- VIX 水平：< 20 正常，20-30 紧张，> 30 恐慌
- SPY 相对 200 日均线位置：上方 → 牛市，下方 → 熊市
- 组合输出：进攻 / 中性 / 防守

**F6: 行业周期**
- 个股营收同比增速的趋势方向（加速 vs 减速）
- 利润率变化方向（扩张 vs 收缩）
- 判断公司处于周期的哪个阶段（复苏/繁荣/衰退/萧条）

### 4.4 因子组合权重

| | 进攻 regime | 中性 regime | 防守 regime |
|---|---|---|---|
| 满仓比例 | 80-100% | 50-80% | 0-30% |
| 选股门槛 | Top 40% | Top 25% | Top 10% |
| 价值权重 (F1+F2) | 40% | 40% | 50% |
| 动量权重 (F3+F4) | 35% | 30% | 20% |
| 行业周期权重 (F6) | 25% | 30% | 30% |

核心逻辑：
- **F5 宏观周期不参与选股评分，而是控制整体仓位水平**
- **F1+F2+F3+F4+F6 加权打分 → 选股排序**
- 防守时提高价值权重，进攻时允许更多动量驱动

## 5. 买卖规则与调仓

### 5.1 调仓频率

月度调仓，每月第一个交易日执行。

### 5.2 调仓流程

```
每月第一个交易日：

1. 更新数据
   - 拉取最新行情（yfinance）
   - 检查是否有新的 10-K/10-Q 发布 → 更新 DCF 估值
   - 计算所有因子得分

2. 宏观 regime 判断
   - 计算 F5 → 决定本月仓位上限（进攻/中性/防守）

3. 选股打分
   - 对股票池中每只股票计算 F1-F4, F6 综合得分
   - 排序，取 top N（N 由 regime 决定）

4. 买入规则
   - 新进入 top N 且当前未持有 → 买入
   - 条件：综合得分 > 阈值 + F3 趋势确认（不抄底下跌刀）

5. 卖出规则（满足任一即卖）
   - 估值回归：市价 ≥ DCF 公允价值（不再低估）
   - 评分跌出：综合得分跌出 top N
   - 止损：持仓亏损 ≥ 15%
   - 技术恶化：跌破 200 日均线 + RSI < 30 继续下行

6. 仓位调整
   - 对持仓按风险平价重新分配权重
   - 单只上限 20%，剩余部分持有现金
```

### 5.3 交易成本假设

| 项目 | 费率 |
|------|------|
| 手续费 | 单边 0.1% |
| 滑点 | 单边 0.05% |
| 单笔总成本 | 约 0.3%（买 + 卖） |

## 6. 回测引擎

### 6.1 回测参数

```
股票池：21 只（Phase 1）→ Screener 动态选股（Phase 2）
起止日期：2016-01 至 2026-01（约 10 年）
初始资金：$100,000
调仓频率：月度
基准：SPY（买入持有）
```

### 6.2 Point-in-Time 数据处理

```
避免前视偏差的关键规则：
- 10-K 数据：使用 filing date + 90 天缓冲
  （例：2023 年 10-K 在 2024-02-15 发布 → 最早 2024-05 才能使用）
- 行情数据：截至调仓日收盘价
- DCF 估值：仅使用"当时能看到的"财务数据重新计算
```

### 6.3 主循环

```python
# 伪代码
for month in months(2016-01, 2026-01):
    # 1. 用当时能看到的数据计算因子（point-in-time）
    factors = compute_factors(universe, as_of=month, pit_data)

    # 2. 宏观 regime
    regime = compute_macro_regime(as_of=month)

    # 3. 选股
    signals = generate_signals(factors, regime)

    # 4. 调仓
    trades = rebalance(portfolio, signals, regime)

    # 5. 记录
    log_trades(trades)
    update_portfolio_value(month)
```

## 7. 绩效指标

### 7.1 收益类
- 总收益率
- 年化收益率（CAGR）
- 月度/年度收益分布

### 7.2 风险类
- 最大回撤（Max Drawdown）
- 年化波动率
- 下行标准差

### 7.3 风险调整类
- 夏普比率（Sharpe Ratio）
- 索提诺比率（Sortino Ratio）
- Calmar 比率（年化收益 / 最大回撤）

### 7.4 对比基准
- vs SPY（买入持有）
- vs 等权持有股票池（不做择时）
- 超额收益 Alpha、Beta

### 7.5 可视化输出
- 净值曲线（策略 vs SPY）
- 月度收益热力图
- 回撤曲线
- 持仓变化时间线

## 8. 文件结构

```
quant/
  data_loader.py      # 加载历史财务+行情，处理 point-in-time
  factors.py           # 因子计算（DCF 折扣、相对估值、技术指标、周期）
  strategy.py          # 买卖规则 + 调仓逻辑
  portfolio.py         # 仓位管理 + 风险平价
  backtest.py          # 回测引擎主循环
  metrics.py           # 绩效指标（夏普、回撤、年化收益等）
  run_backtest.py      # 入口脚本
```

## 9. 实施阶段

### Phase 1：跑通完整流程（用 21 只股票）
- 实现数据层 + 因子层 + 策略层 + 回测引擎
- 输出基本绩效指标
- 与 SPY 对比

### Phase 2：接入 Screener 动态选股
- 每月从 screener 结果中动态选股
- 处理股票池变化（新进/退出）

### Phase 3：调参优化 + 因子分析
- 单因子回测：分别测试每个因子的独立贡献
- 参数敏感性分析：止损比例、调仓频率、因子权重
- 对比不同因子组合的效果
